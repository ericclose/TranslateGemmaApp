# TranslateGemmaApp 测试说明 (v1.2.1-beta)

这份文档旨在指导您对当前版本的 TranslateGemmaApp 进行全面的功能验证。

## 1. 安装测试
- [ ] **DMG 挂载**：打开 `TranslateGemmaApp-arm64-v1.2.1-beta.dmg`，确认是否能看到“应用程序”快捷方式。
- [ ] **拖拽安装**：尝试将 App 拖入右侧的 `Applications` 图标，确认安装流程是否顺畅。
- [ ] **首次运行**：打开 App，确认是否因无本地模型而自动弹出 **Model Dashboard**。

## 2. 模型管理测试 (Model Dashboard)
- [ ] **模型下载**：点击任一模型的 `Download` 按钮。
    - [ ] 观察进度条是否随真实下载进度更新。
    - [ ] 下载完成后，状态是否变为绿色的勾选图标。
- [ ] **Reveal in Finder**：点击下载完成模型旁的文件夹图标，确认是否能准确跳转到 Hugging Face 的本地缓存路径。
- [ ] **持久化检查**：彻底退出 App 并重新打开，进入 Dashboard 确认已下载模型的状态是否依然为“已完成”。

## 3. 基础翻译功能测试
- [ ] **模型切换**：在主界面顶部下拉菜单中，确认是否能看到所有已下载的模型并进行切换。
- [ ] **文本输入**：在左侧区域输入一段简单的英文文本（如 "Hello, how are you today?"）。
- [ ] **执行翻译**：点击 `Translate` 按钮。
    - [ ] 观察按钮是否进入 `Translating...` 加载状态。
    - [ ] 观察右侧区域是否出现真实的模型生成文本。
- [ ] **语言切换**：切换目标语言（如从 Chinese 换成 Japanese），再次翻译，确认输出语种是否变化。

## 4. 文件格式翻译测试 (核心功能)
- [ ] **SRT 字幕测试**：
    - [ ] 点击 `Import File` 并选择一个 `.srt` 文件。
    - [ ] 点击 `Translate`，翻译完成后点击 `Export`。
    - [ ] **验证要点**：打开导出的文件，确认**时间轴、序号**是否保持原样，仅对话内容被翻译。
- [ ] **Markdown 测试**：
    - [ ] 导入一个包含代码块、标题和链接的 `.md` 文件。
    - [ ] 翻译并导出。
    - [ ] **验证要点**：确认**代码块和 Markdown 语法标识**（如 `#`, `**`, `[link]()`）是否被破坏。
- [ ] **ASS/VTT 测试**：参照 SRT 测试流程，确认对应的字幕格式结构是否完整保留。

## 5. UI 细节与逻辑测试
- [ ] **清除功能**：导入文件后，点击 `Clear` 按钮，确认 `importedFileURL` 状态被重置，输入框恢复为空。
- [ ] **导出建议**：在导出已翻译文件时，确认默认文件名是否以 `translated_` 为前缀。
- [ ] **选择记忆**：切换选中模型 -> 退出 App -> 重新进入，确认顶部 Picker 选中的是否依然是退出前的模型。

## 6. 异常情况处理
- [ ] **离线状态**：断开网络后，尝试翻译已下载的模型，确认是否能完全脱离网络运行。
- [ ] **未下载模型**：如果没有下载任何模型，尝试点击 `Translate`，确认是否会正确引导至 Dashboard。

---
**测试反馈建议**：如果您在测试中发现翻译结果包含模型自言自语（如 "Sure, I can help with that"）或格式错乱，请记录对应的 Prompt 格式，我们将进一步微调。
